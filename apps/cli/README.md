## 🎯 What is Vibrant?

Vibrant detects **"vibecoding"** - code that looks like it was generated by AI without proper review. It catches:

- 🐛 **Bugs**: Empty catch blocks, unhandled promises, null access
- 🔒 **Security issues**: Hardcoded secrets, SQL injection, XSS
- 🤖 **AI telltales**: console.log in production, TODO comments, generic names
- 🏗️ **Architecture problems**: Deeply nested code, functions doing too much

## 🚀 Quick Start

### Basic Analysis

```bash
# Analyze current directory
vibrant .

# Analyze specific file
vibrant src/api.ts

# Analyze with glob pattern
vibrant "src/**/*.ts"
```

### AI-Powered Analysis

```bash
# Enable AI analysis
vibrant . --ai

# With specific provider
vibrant . --ai --provider openrouter
```

### Auto-fix Issues

```bash
# Fix auto-fixable issues
vibrant . --fix
```

## 🔮 AI Providers

Vibrant supports multiple AI providers. Set your API key and go:

| Provider | Environment Variable | Free Tier |
|----------|---------------------|-----------|
| **OpenRouter** | `OPENROUTER_API_KEY` | ✅ Free models available |
| **OpenAI** | `OPENAI_API_KEY` | ❌ Paid |
| **Gemini** | `GOOGLE_GENERATIVE_AI_API_KEY` | ✅ Free tier |
| **Claude** | `ANTHROPIC_API_KEY` | ❌ Paid |
| **Ollama** | `OLLAMA_HOST` | ✅ Local, free |

### Setup

```bash
# Option 1: Environment variable
export OPENROUTER_API_KEY="sk-or-..."

# Option 2: .env file
echo 'OPENROUTER_API_KEY="sk-or-..."' > .env

# Option 3: Config file
vibrant init
# Edit vibrant.config.js
```

## 🔧 Configuration

Create `vibrant.config.js`:

```javascript
module.exports = {
  // Directories to ignore
  ignore: ['node_modules', '.git', 'dist', '.next'],
  
  // Output format
  format: 'pretty', // 'pretty' | 'compact' | 'json' | 'plan'
  
  // Default AI provider
  provider: 'openrouter',
  
  // Custom rules (optional)
  rules: {
    'console-log-debugging': 'warn',
    'hardcoded-credentials': 'error',
  }
};
```

## 🧠 How It Works

### 1. Smart Summarizer

Vibrant uses a **Smart Summarizer** that:

- ✅ Detects suspicious patterns (console.log, TODOs, secrets)
- ✅ Preserves suspicious lines with context
- ✅ Compresses clean code to signatures only
- ✅ Saves ~50-60% tokens while maintaining accuracy

### 2. AI Analysis

The AI receives:

```
## src/api.ts

Suspicious:
L24|const API_KEY = "sk-abc";  // secret
L45|console.log("debug");  // console

Functions:
L10|async fn fetchData(url)
L60|fn processResult(data)
```

### 3. Pattern Detection

Vibrant detects:

**Security:**
- Hardcoded API keys, passwords, tokens
- SQL injection, XSS vulnerabilities
- `eval()`, `innerHTML` with user input

**Bugs:**
- Empty catch blocks
- Unhandled promise rejections
- Null/undefined without checks

**Code Quality:**
- `console.log` in production paths
- `// TODO: implement` comments
- Generic names: `data`, `result`, `temp`
- `any` type, `@ts-ignore`

## 🛠️ CLI Commands

```bash
vibrant [path]           # Analyze code (default: .)
vibrant init             # Create config file
vibrant rules            # List available rules
vibrant --help           # Show help
vibrant --version        # Show version
```

### Options

| Option | Description |
|--------|-------------|
| `--ai` | Enable AI-powered analysis |
| `-p, --provider <name>` | AI provider (openai, claude, gemini, ollama, openrouter) |
| `-f, --format <type>` | Output format (pretty, compact, json, plan) |
| `--fix` | Auto-fix issues |
| `--ignore <patterns>` | Comma-separated ignore patterns |
