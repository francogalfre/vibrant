## What is Vibrant?

Vibrant is a CLI tool that detects **vibecoded** patterns — code that looks like it was generated by AI without proper review. It catches shortcuts, placeholders, security issues, and anti-patterns that slip through when developers use AI assistants.

## Quick Start

```bash
# No installation needed
npx vibrant .

# Enable AI-powered analysis
npx vibrant . --ai

# Auto-fix issues
npx vibrant . --fix
```

## Features

- **Static Analysis** — 15+ rules to detect common patterns
- **AI-Powered Analysis** — Deep pattern detection with LLMs
- **Auto-Fix** — Automatically fix fixable issues
- **Multiple Providers** — OpenAI, Claude, Gemini, Ollama, OpenRouter
- **Multiple Formats** — Pretty, compact, JSON, plan
- **AI Telltales** — Detect emojis in comments, excessive TODOs, magic numbers
- **Fun Feedback** — Humorous messages when issues are found

## Commands

```bash
vibrant [path]           # Analyze code (default: current directory)
vibrant init             # Create config file
vibrant rules            # List available rules
vibrant --help           # Show help
vibrant --version        # Show version
```

## Options

| Option | Description |
| --- | --- |
| `--ai` | Enable AI-powered analysis |
| `-p, --provider <name>` | AI provider: openai, claude, gemini, ollama, openrouter |
| `-f, --format <type>` | Output: pretty, compact, json, plan |
| `--fix` | Auto-fix issues |
| `--ignore <patterns>` | Ignore patterns (comma-separated) |

## AI Providers

Set an environment variable for your preferred provider:

| Provider | Variable | Info |
| --- | --- | --- |
| **OpenRouter** | `OPENROUTER_API_KEY` | Free models available |
| **OpenAI** | `OPENAI_API_KEY` | GPT-4o-mini |
| **Gemini** | `GOOGLE_GENERATIVE_AI_API_KEY` | Free tier |
| **Claude** | `ANTHROPIC_API_KEY` | Claude 3 Haiku |
| **Ollama** | `OLLAMA_HOST` | Local, free |

```bash
# Example
export OPENROUTER_API_KEY="sk-or-..."
vibrant . --ai

# Or specify provider
vibrant . --ai --provider gemini
```

## Detected Patterns

### Security

| Pattern | Description |
| --- | --- |
| `hardcoded-credentials` | API keys, passwords, tokens in code |
| `no-sql-injection` | SQL injection vulnerabilities |
| `no-unsafe-inner-html` | XSS via innerHTML |

### Bugs

| Pattern | Description |
| --- | --- |
| `empty-catch-block` | Catch blocks that swallow errors |
| `empty-function-body` | Functions without implementation |
| `unimplemented-error` | `throw new Error("not implemented")` |
| `no-unreachable` | Unreachable code |
| `no-ex-assign` | Assigning to exception variable |
| `use-isnan` | Using === with NaN |

### Code Quality

| Pattern | Description |
| --- | --- |
| `console-log-debugging` | Leftover console statements |
| `no-explicit-any` | TypeScript `any` usage |
| `no-await-in-loop` | Sequential awaits in loops |

### AI Telltales

| Pattern | Description |
| --- | --- |
| `ai-comment-emojis` | Emojis in code comments |
| `ai-todo-comments` | Excessive TODO/FIXME comments |
| `magic-numbers` | Unexplained numeric literals |

## Ignoring Issues

### Ignore Specific Lines

```typescript
// Vibrant ignore - ignores this line
const apiKey = "secret";

// Vibrant ignore-next-line - ignores the next line
const secret = "password";
```

### Ignore Files

```javascript
// vibrant.config.js
module.exports = {
  ignore: ["node_modules", ".git", "dist", "*.test.ts"]
};
```

## Configuration

Create `vibrant.config.js`:

```javascript
module.exports = {
  ignore: ["node_modules", ".git", "dist"],
  format: "pretty",
  // provider: 'openrouter',
};
```

Or run `vibrant init` to create one.

## Installation

```bash
# npx (no installation)
npx vibrant .

# Bun (recommended)
bun install -g vibrant-cli

# npm
npm install -g vibrant-cli

# From source
git clone https://github.com/francogalfre/vibrant.git
cd vibrant/apps/cli
bun install
bun run build
```
